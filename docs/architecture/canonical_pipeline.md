# Canonical Pipeline

Status: canonical
Confidence: high

## Purpose

This document defines the **canonical Hyperflow pipeline** at the MVP Runtime Baseline stage.

## Canonical cycle

The canonical Hyperflow cycle is:

`🌈💎🔥🧠🔀⚡`

## Canonical meaning

- `🌈` = perceive
- `💎` = extract essence / core
- `🔥` = set direction
- `🧠` = synthesize
- `🔀` = generate / compare options
- `⚡` = choose

## Canonical logic

The final part of the cycle must be:

`🧠 -> 🔀 -> ⚡`

not:

`🧠 -> ⚡ -> 🔀`

This means:
- synthesize first
- compare options second
- choose last

## Legacy order

The old order `🌈💎🔥🧠⚡🔀` is not canonical.
It may remain supported only as a deprecated legacy alias for backward compatibility.

## Canonical runtime semantics

At runtime, the canonical operation sequence is:

- `perceive`
- `extract_core`
- `set_direction`
- `synthesize`
- `generate_options`
- `choose`

## Alignment rule

The canonical pipeline is implemented only when the following all agree:
- runtime code
- parser/control semantics
- canonical config
- tests
- docs
