Use these paths as the active documentation surface for the canonical MVP repo:

- `docs/architecture/definition_of_done.md` — baseline completion bar
- `docs/architecture/canonical_core.md` — current canonical core and scope boundaries
- `docs/contracts/runtime_contract.md` — current runtime contract
- `docs/contracts/api_contract.md` — public API surface for the baseline
- `docs/architecture/canonical_pipeline.md` — locked canonical cycle and phase meanings
- `docs/architecture/authority_map.md` — source-of-truth and ownership map
- `docs/architecture/merge_decisions.md` — durable decision log
- `docs/migration/` — donor mapping, archive policy, and consolidation notes

Historical overlays, narrative snapshots, and superseded planning bundles were moved to `archive/docs/`.

This is the only active documentation index under `docs/`.

Boundary rules for active docs:
- archive material is reference-only
- generated runtime artifacts are outputs, not canonical source truth
- future-layer framework/platform material requires an explicit promotion decision before gaining canonical authority
